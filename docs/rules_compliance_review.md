# ルールブック準拠状況の調査レポート (Rules Compliance & Bug Review) - 【対応完了】

ルールブック（`ローグライクハーフ 基本ルール　ver.5.1.pdf`）の記述と現在のSPA実装を突き合わせ調査した結果、ルールと異なっていた挙動や不具合について、**すべての対応が完了しました**。

---

## 🛠️ 対応完了した項目一覧

### ✅ 1. 鎧（防具）の所持制限 (Rule 27) - 【修正済】
* **ルール:** 「着用している以外の鎧は持ち歩けない」という規定。
* **対応内容:** 
  - 街の市場（`App.vue` の `buyArmor`）において、すでに鎧を所持している場合は、古い鎧が自動的に安全に破棄され、新しい鎧と交換（自動装備）される仕組みを導入しました。
  - バックパック内に複数の鎧が混在して持ち歩けるバグを解消しました。

### ✅ 2. 生命力最大値（`lifeMax`）の破損バグの修正 (Rule 30/15) - 【修正済】
* **ルール:** 防具や盾を装備すると `modLife` ボーナスが最大生命力に加算され、外すと両方が引き下がります。
* **対応内容:**
  - 装備の脱着時に最大生命力を初期値（4）へ強制リセットするハードコーディングを廃止しました。
  - `lifeMax` と `lifeCurrent` をアイテムの `modLife` の相対値で増減するようにロジックを修正しました。これにより、成長したベース最大HPを破損することなく装備の着脱が可能になりました。
  - ゲーム開始時およびリトライ時にも、初期装備の `modLife`（例: 布鎧なら生命最大値+1）が正しく計算（初期HPが5や6になるなど）されて反映されるようになりました。

### ✅ 3. 初期修得魔法・奇跡の制限超過 (Rule 18/20) - 【修正済】
* **ルール:** 修得できる魔法・奇跡の数は魔術点の半分（初期魔術点2なら1つ）である規定。
* **対応内容:**
  - キャラクター初期作成時および再挑戦（リトライ）時に、初期修得数を 1 つ（魔術ルートなら「気絶」、幸運ルートなら「防衛」）に制限し、ルールに完全準拠させました。

### ✅ 4. クリティカル発生時の敵撃破処理バイパスバグの修正 - 【修正済】
* **不具合内容:** 攻撃ロールで出目6（クリティカル）を引いた際に追加攻撃権の早期リターンが走り、敵のHPが0以下になっても即座に撃破判定が処理されない不具合。
* **対応内容:**
  - 敵のHPが0以下になったかどうかの撃破判定を、クリティカルによる追加ターンの早期リターンよりも**前**に実行するようにコード順序を修正しました。クリティカルでトドメを刺した瞬間に正常に敵の撃破が反映されます。

### ✅ 5. 従者魔術師（Mage）の魔術行使の実装 (Rule 33) - 【修正済】
* **ルール:** 魔術師の従者は魔術点1を持ち、基礎的な魔術（炎球など）を行使できる規定。
* **対応内容:**
  - 従者の行動フェーズにおいて、従者魔術師が生存しており、かつ魔術点（MP）が残っている場合、戦闘時に自動的に呪文 **[炎球]** を唱えるAIロジックを実装しました。
  - 魔法は魔術点を消費し、対象モンスターへ魔術判定ロールを行って1点のスペルダメージを適切に与えます。魔法を使い切った後は、通常通り軽い武器（攻撃力修正-1）での援護攻撃へシフトします。

### ✅ 6. ダイスロールの「判定」と「表・数値決定」の明確な分離 (Rule 8) - 【修正済】
* **ルール:** クリティカル（出目6で自動成功）やファンブル（出目1で自動失敗）の概念は、能力値や技量点を用いて成否を競う「判定ロール（Check Rolls）」にのみ適用されます。戦利品（金貨・宝石・魔法の宝物など）を獲得する「宝物表ロール」やトラップのダメージ決定など、表を参照するためのロールには適用されません。
* **対応内容:**
  - `rollD6` 関数を拡張し、判定ロールであるかどうかを示す引数 `isCheck: boolean` を追加しました。
  - 攻撃・防御・罠解除・スペル詠唱などの「判定ロール」に対してのみ `rollD6(true)` を呼び出し、UI（ダイストレイ）上で「★クリティカル！」「💀ファンブル！」の演出やログ出力が適用されるようにしました。
  - 戦利品の金額決定や宝石のサイズ決定、宝物表のダイスロールなど、単純な数値決定・表参照ロールは `rollD6(false)` とし、ファンブルやクリティカルとしての誤判定・誤演出が発生しないように完全に分離しました。
