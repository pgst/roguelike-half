## ステップ1：シンフリーサーバー側での準備

まずは、GitHub Actionsからサーバーにアクセスできるように設定を整えます。

1. **SSH設定を有効にする**
* シンフリーサーバーの「サーバーパネル」にログインします。
* 「SSH設定」を開き、設定を「ON」に変更します。


2. **公開鍵・秘密鍵ペアの生成**
* 同じく「SSH設定」画面の「公開鍵を登録（自動生成）」などから鍵ペアを作成します。
* 登録すると、自動的に秘密鍵ファイル（`ユーザーID.key`）がローカルPCにダウンロードされます。この中身を後で使います。


3. **【超重要】国外アクセス制限を「OFF」にする**
* SSH設定画面にある「国外アクセス制限」の設定を「OFF」にしてください。
* ※これを忘れると、GitHub Actions（海外サーバー）からの接続がすべてブロックされてデプロイが失敗します。


4. **接続情報の確認**
* サーバーパネルの「サーバー情報」等で、以下の項目を確認しておきます。
* **ホスト名**（例: `sv*.cloudfree.ne.jp`）
* **ユーザー名**（サーバーID）
* **ポート番号**：**`10022`** （シンフリーサーバーのSSHポートは一律10022です）

## ステップ2：GitHubに秘密情報を登録（Secrets）

ダウンロードした秘密鍵やサーバーの情報を、GitHubのリポジトリに安全に登録します。

1. GitHubの対象リポジトリを開き、「Settings」＞「Secrets and variables」＞「Actions」の順にクリックします。
2. 「New repository secret」をクリックし、以下の3つ（または4つ）の値を登録してください。

| 名前 | 値の例・説明 |
| --- | --- |
| `SFTP_HOST` | サーバーのホスト名（例: `sv123.cloudfree.ne.jp`） |
| `SFTP_USERNAME` | サーバーのユーザー名（サーバーID） |
| `SFTP_PRIVATE_KEY` | ダウンロードした `.key` ファイルをテキストエディタで開き、**中身を丸ごとすべてコピー**して貼り付けます。 |

## ステップ3：GitHub Actionsのワークフローファイルを作成

リポジトリのルート直下に `.github/workflows/deploy.yml` というファイルを作成し、以下のコードを記述してコミット＆プッシュします。

今回は、純粋なSFTP転送でエラーが起きにくい `wlixcc/SFTP-Deploy-Action` を使用した例を紹介します。

```yaml
name: Deploy Vue App

on:
  push:
    branches:
      - main # デプロイのトリガーにするブランチ（masterの場合は変更してください）

jobs:
  deploy:
    runs-on: ubuntu-latest

    steps:
      # 1. コードのチェックアウト
      - name: Checkout repository
        uses: actions/checkout@v4

      # 2. Node.js環境のセットアップ
      - name: Set up Node.js
        uses: actions/setup-node@v4
        with:
          node-version: 20 # お使いのVueのバージョンに合わせて調整してください
          cache: 'npm'

      # 3. 依存関係のインストールとビルド
      - name: Install dependencies
        run: npm ci

      - name: Build Vue App
        run: npm run build # ViteやVue CLIで設定されたビルドコマンド

      # 4. シンフリーサーバーへのSFTPデプロイ
      - name: Deploy via SFTP
        uses: wlixcc/SFTP-Deploy-Action@v1.2.4
        with:
          username: ${{ secrets.SFTP_USERNAME }}
          host: ${{ secrets.SFTP_HOST }}
          port: 10022 # シンフリーサーバー専用のSSHポート
          ssh_private_key: ${{ secrets.SFTP_PRIVATE_KEY }}
          local_path: './dist/*' # Vueのビルド成果物が入るディレクトリ
          # リモート（サーバー側）のパス。初期ドメインの場合は以下のような構成になります
          remote_path: '/home/${{ secrets.SFTP_USERNAME }}/${{ secrets.SFTP_USERNAME }}.cloudfree.jp/public_html'

```

> [!WARNING]
> **`remote_path`（アップロード先）について**
> 上記の例は、初期ドメイン（`ユーザーID.cloudfree.jp`）のドキュメントルートを想定しています。もし独自ドメインを設定していて、その配下にアップロードしたい場合は、`.../public_html` の部分を実際のフォルダ構成に合わせて書き換えてください。

## 💡 Vue（Viteなど）側の注意点

もしアプリをドメインのトップ（`https://〜.cloudfree.jp/`）ではなく、サブディレクトリ（例: `https://〜.cloudfree.jp/my-app/`）に配置する場合は、Vue側（Vite等）のベースパスの設定が必要です。

**Vite（`vite.config.js`）の場合の例:**

```javascript
export default defineConfig({
  base: './', // または '/my-app/'
  plugins: [vue()],
})

```

これを設定しておかないと、デプロイ後に「画面が真っ白で、CSSやJSが404エラーで読み込めない」という現象が起きるので注意してください。
